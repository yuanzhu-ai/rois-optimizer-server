# 优化引擎调度工具需求文档

## 1. 项目概述

### 1.1 项目背景
公司开发了多种可执行优化器（如PO、RO、TO等），目前需要一个统一的调度工具来管理这些优化器的部署和执行，支持多机部署，并提供统一的状态管理和监控能力。

### 1.2 项目目标
构建一个轻量级的优化引擎调度Server，提供以下核心功能：
- 统一管理不同类型的优化器
- 支持多机部署
- 提供优化进度和状态的查询
- 支持优化器的启动和杀停
- 跨平台支持（Linux和Windows）
- 自动化文件管理
- 提供RESTful API接口

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 优化器管理
- **功能描述**：支持管理多种类型的优化器（PO、RO、TO等）
- **详细需求**：
  - 支持配置不同类型优化器的执行路径和参数
  - 支持在不同平台（Linux/Windows）上使用对应的脚本执行（sh/bat）
  - 支持多实例同时运行，每个实例使用独立的工作目录

#### 2.1.2 调度控制
- **功能描述**：提供优化器的启动、停止控制
- **详细需求**：
  - 支持通过API启动优化任务
  - 支持临时杀停正在执行的优化任务
  - 支持根据请求内容调用不同的Server地址

#### 2.1.3 状态管理
- **功能描述**：统一管理优化任务的状态和进度
- **详细需求**：
  - 提供优化进度查询接口
  - 提供优化完成状态返回（成功/失败）
  - 不用连接数据库，使用内存或文件存储状态信息

#### 2.1.4 文件管理
- **功能描述**：自动化管理优化过程中产生的文件
- **详细需求**：
  - 优化完成后将文件移动至Finished文件夹
  - 隔天将Finished文件夹中的文件移动至archive文件夹并压缩
  - 定期清理archive文件夹内超过指定日期的压缩文件

#### 2.1.5 接口集成
- **功能描述**：与现有系统接口集成
- **详细需求**：
  - 支持使用token进行权限认证
  - 调用现有POST接口获取input.gz文件
  - 调用现有POST接口回传output.gz文件

### 2.2 API需求

#### 2.2.1 优化任务管理接口
- **启动优化任务**：POST /api/optimize/start
- **停止优化任务**：POST /api/optimize/stop/{task_id}
- **查询优化任务状态**：GET /api/optimize/status/{task_id}
- **查询优化任务进度**：GET /api/optimize/progress/{task_id}

#### 2.2.2 系统管理接口
- **获取系统信息**：GET /api/system/info
- **获取优化器列表**：GET /api/optimizers
- **获取运行中任务**：GET /api/tasks/running

## 3. 非功能需求

### 3.1 性能需求
- **响应时间**：API响应时间不超过500ms
- **并发处理**：支持同时处理至少10个优化任务
- **资源占用**：内存占用不超过512MB，CPU使用率不超过50%

### 3.2 可靠性需求
- **故障恢复**：服务重启后能恢复之前的任务状态
- **错误处理**：提供详细的错误信息和日志
- **稳定性**：服务稳定运行时间不低于7天

### 3.3 安全性需求
- **认证**：使用token进行API认证
- **授权**：基于角色的访问控制
- **数据安全**：传输过程中的数据加密

### 3.4 可扩展性需求
- **多机部署**：支持在多台服务器上部署
- **水平扩展**：支持通过负载均衡扩展服务能力
- **插件架构**：支持添加新的优化器类型

## 4. 技术架构

### 4.1 技术栈
- **语言**：Python
- **Web框架**：FastAPI（推荐）或Flask
- **任务管理**：Celery或自定义任务队列
- **文件处理**：Python标准库
- **跨平台支持**：使用os模块适配不同平台

### 4.2 架构设计
- **API层**：处理HTTP请求，提供RESTful接口
- **业务逻辑层**：处理调度逻辑，管理优化任务
- **执行层**：负责执行优化器，监控执行状态
- **文件管理层**：处理文件的移动、压缩和清理
- **配置管理层**：管理系统配置和优化器配置

### 4.3 部署方案
- **Linux**：使用sh脚本启动服务
- **Windows**：使用bat脚本启动服务
- **配置文件**：使用YAML或JSON格式的配置文件

## 6. 风险评估

### 6.1 技术风险
- **跨平台兼容性**：不同平台的脚本执行差异
- **任务调度**：多任务并发执行的资源管理
- **文件处理**：大量文件的处理和存储

### 6.2 业务风险
- **接口集成**：与现有系统的接口兼容性
- **权限管理**：token认证的安全性
- **系统稳定性**：长时间运行的稳定性

- **优化器**：指公司开发的各种可执行优化工具，如PO、RO、TO等

- **Server**：指本调度工具服务

- **output.gz**：优化器的输出文件
